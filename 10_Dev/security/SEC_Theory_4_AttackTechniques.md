# SEC_Theory_4: 공격 기법 개요 (피싱/익스플로잇/권한상승/측면이동/C2)

## 🧠 핵심 개념 한 줄 요약
실제 해킹은 한 방에 끝나지 않고 **피싱(침투) → 익스플로잇(실행) → 권한 상승 → 측면 이동 → C2 통신(원격 조종)** 순서의 여러 단계를 거치는 프로세스다.

## 📦 핵심 개념 상세

### 전체 흐름
```
피싱 (침투) → 익스플로잇 (실행) → 권한 상승 → 측면 이동 → C2 통신 (원격 조종)
```
이 순서는 MITRE ATT&CK가 공격을 단계별로 분류하는 방식과도 맞닿아 있음.

### 1️⃣ 피싱 (Phishing) — MITRE T1566
- 비유: 가짜 은행 직원이 전화해서 계좌번호를 물어봄
- 신뢰할 만한 대상(은행/IT팀/상사)을 사칭해 정보를 빼내거나 악성 파일을 실행시키는 기법
- 초기 침투(Initial Access)의 가장 흔한 경로 — 실제 침해사고의 90% 이상이 피싱에서 시작된다는 통계도 있음
- 예: "인사팀 급여명세서.xlsx" 위장 이메일 → 매크로 실행 → 악성코드 다운로드

### 2️⃣ 익스플로잇 (Exploit) — MITRE T1203
- 비유: 고장난 창문 잠금장치를 실제로 여는 행위
- 소프트웨어의 Vulnerability(취약점)를 실제로 악용해 원하는 코드를 실행시키는 것
- 피싱과의 차이: 피싱은 "사람"을 속이는 것, 익스플로잇은 "소프트웨어 결함"을 이용하는 것 (종종 결합됨 — 피싱 첨부파일이 PDF 리더 취약점을 익스플로잇)
- 예: 버퍼 오버플로우(Buffer Overflow) 취약점으로 정상 흐름을 깨고 공격자 코드 실행

### 3️⃣ 권한 상승 (Privilege Escalation) — MITRE T1068
- 비유: 청소부로 위장 침입했다가 관리자 마스터키를 훔침
- 낮은 권한(일반 사용자)으로 침투 후 관리자(Admin/Root/SYSTEM) 권한을 획득하는 과정
- 필요 이유: 시스템 파일 변경, 다른 계정 접근, 보안 프로그램 무력화 등은 관리자 권한 필요
- 기법 예: 취약한 서비스 권한 상속 악용, UAC 우회, 커널 취약점 이용 SYSTEM 권한 탈취

### 4️⃣ 측면 이동 (Lateral Movement) — MITRE T1021
- 비유: 1층 침입 도둑이 마스터키로 2층, 3층, 다른 사무실까지 옮겨 다니며 뒤짐
- 침투한 PC 한 대에서 그치지 않고 네트워크 내 다른 PC/서버로 옮겨 다니며 범위 확장
- 이유: 처음 침투한 PC에 원하는 정보가 없을 수 있음 — 진짜 목표(도메인 컨트롤러, DB 서버 등)에 도달하기 위한 중간 과정
- 기법 예: 훔친 계정 정보로 다른 PC 원격 접속(RDP, SMB), Pass-the-Hash(비밀번호 해시값을 그대로 이용해 인증 우회)

### 5️⃣ C2 통신 (Command and Control) — MITRE T1071 / T1071.004(DNS)
- 비유: 침투한 스파이가 본부와 몰래 무전기로 계속 교신하며 다음 지시를 받음
- 감염된 PC(봇/에이전트)가 공격자의 외부 서버(C2 서버)와 지속 통신하며 명령을 받고 결과를 전송
- 이 통신 채널이 끊기면 공격자는 더 이상 조종 불가 → **C2 통신 탐지는 EDR/네트워크 보안의 핵심 방어선**
- 은닉 기법: 정상 트래픽(HTTPS, DNS)으로 위장 (예: DNS 터널링)
- 암호학(대칭키/비대칭키, 해시)이 필요한 이유: C2 통신 대부분 암호화되어 있어 탐지 시스템이 내용을 볼 수 없고 **패턴(통신 주기, 목적지, 크기)**으로 추론해야 함

## 📊 전체 공격 흐름 정리

| 단계 | 목적 | 비유 | MITRE ID |
|---|---|---|---|
| 피싱 | 초기 침투 | 가짜 은행원 사칭 | T1566 |
| 익스플로잇 | 코드 실행 | 고장난 창문 열기 | T1203 |
| 권한 상승 | 더 높은 권한 획득 | 마스터키 훔치기 | T1068 |
| 측면 이동 | 침투 범위 확장 | 다른 층으로 이동 | T1021 |
| C2 통신 | 원격 조종 유지 | 무전기로 본부 교신 | T1071 |

## 🔗 EDR과의 연관성
EDR이 감시하는 이벤트들은 이 5단계 각각에 대응된다.

- **피싱 단계**: 이메일 첨부파일 실행 시 자식 프로세스로 스크립트/매크로 실행 여부 감시 (예: Word.exe → cmd.exe/powershell.exe 실행은 매우 의심스러운 패턴)
- **익스플로잇 단계**: 프로세스의 비정상 메모리 접근이나 크래시 후 예상치 못한 코드 실행 감시
- **권한 상승 단계**: 프로세스 토큰(Token) 권한의 갑작스러운 변경, 알려진 권한 상승 취약점 패턴 사용 여부 감시
- **측면 이동 단계**: 짧은 시간 내 여러 PC로의 인증 시도, PsExec/WMI 등 원격 실행 도구의 비정상적 사용 감시
- **C2 통신 단계**: 네트워크 연결 패턴 분석 — 일정 주기로 반복되는 통신("Beaconing") 탐지

> EDR 설계자는 "이 5단계 중 어느 단계에서, 어떤 행위 로그를 수집해야 탐지할 수 있는가"를 고민하는 사람이다.

## ⚠️ 자주 하는 오해/실수
- 피싱과 익스플로잇을 같은 개념으로 혼동하기 쉬움 — 피싱은 "사람"을 속이는 것, 익스플로잇은 "소프트웨어 결함"을 이용하는 것으로 대상이 다름
- 공격이 항상 이 5단계를 순서대로, 전부 거치는 건 아님 — 상황에 따라 생략되거나 반복될 수 있음 (예: 측면 이동 후 또 다른 권한 상승이 필요할 수 있음)
- C2 통신 탐지를 "내용을 복호화해서 보는 것"으로 오해하기 쉬운데, 실제로는 대부분 암호화되어 있어 **패턴 기반 추론**이 핵심
