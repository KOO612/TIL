# SEC_OSInternals_4_ProcessInjectionConcepts

## 🧠 핵심 개념 한 줄 요약
프로세스 인젝션(MITRE ATT&CK T1055)은 공격자가 새 프로세스를 만드는 대신 이미 신뢰받는 정상 프로세스 안에서 코드를 실행시켜 EDR/백신의 탐지를 회피하는 기법이며, DLL 인젝션과 프로세스 할로잉이 대표적이다.

## 📦 핵심 개념 상세

### 비유: 인젝션 = 남의 사무실에 몰래 들어가 일하기
공격자가 자기 회사(악성 프로세스)를 새로 차리지 않고, 이미 신뢰받는 회사(정상 프로세스, 예: notepad.exe)의 사무실에 몰래 들어가 그 이름으로 일을 시킨다. 보안 카메라(EDR)는 "notepad.exe가 하는 일"로 기록하므로 의심을 덜 받는다.

### 1. DLL 인젝션 (DLL Injection)
```
1. 대상 프로세스에 OpenProcess()로 핸들 획득
2. VirtualAllocEx()로 대상 프로세스 메모리에 공간 할당
3. WriteProcessMemory()로 "로드할 DLL 경로 문자열" 써넣기
4. CreateRemoteThread()로 대상 프로세스 안에서
   LoadLibrary(그 경로)를 실행하는 원격 스레드 생성
5. 대상 프로세스가 악성 DLL을 "스스로" 로드하게 됨
```
비유: 남의 사무실 우편함(메모리)에 "이 서류(DLL) 좀 읽어줘"라는 쪽지를 몰래 넣고, 그 사무실 직원(원격 스레드)이 실제로 읽게 만드는 것.

### 2. 프로세스 할로잉 (Process Hollowing)
```
1. 정상 프로세스를 "일시정지 상태"로 생성 (CREATE_SUSPENDED)
2. 그 프로세스의 원래 코드(이미지)를 메모리에서 지움 ("속을 파냄")
3. 그 자리에 악성 코드 이미지를 새로 매핑
4. 프로세스를 재개(Resume) → 겉보기엔 정상 프로세스, 속은 악성코드
```
비유: 정상 회사 간판(프로세스 이름, PID)은 그대로 두고, 건물 안 내용물(실행 이미지)만 통째로 바꿔치기.

## 🔗 EDR과의 연관성
- **DLL 인젝션 탐지**: `CreateRemoteThread()` 호출 자체를 후킹/감시. 다른 프로세스에 원격 스레드를 만드는 경우는 드물어 의심 신호로 취급.
- **프로세스 할로잉 탐지**: 프로세스 메모리 이미지가 디스크상 원래 파일과 일치하는지 비교(PEB의 이미지 베이스 주소 vs 실제 파일 해시). 불일치 시 할로잉 의심.
- **RWX 메모리 감지**: 인젝션된 코드가 실행되려면 결국 실행 권한 있는 메모리가 필요 → 앞서 배운 "쓰기+실행 동시 권한" 탐지가 여기서도 핵심 무기.
- 면접 단골 질문: "DLL 인젝션이 무엇이고 어떻게 탐지하나요?" — 반드시 완전히 이해하고 설명 가능해야 함.

## ⚠️ 자주 하는 오해/실수
- "인젝션은 무조건 악성이다" → 틀림. 게임 모드, Discord 오버레이 등 정상 소프트웨어도 DLL 인젝션 기법을 사용. EDR은 "인젝션 여부"뿐 아니라 "누가, 어떤 맥락에서" 했는지까지 봐야 오탐(False Positive)을 줄일 수 있음.
- "프로세스 할로잉은 PID만 보면 바로 알 수 있다" → 틀림. PID·프로세스 이름은 그대로라서 겉보기 정보만으론 구분 불가. 메모리 이미지 검증이 필요.
