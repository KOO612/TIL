# SEC_EDR_2 — Windows API 후킹 (IAT 후킹 / 트램폴린 후킹)

## 🧠 핵심 개념 한 줄 요약
API 후킹은 **프로그램이 함수를 호출하는 순간을 가로채는 기법**이며, EDR에게는 "공격자가 숨는 방법"이자 동시에 "EDR이 감시하는 방법"이라는 이중적 의미를 가진다.

---

## 📦 핵심 개념 상세

### 1. 함수 호출의 기본 구조 — IAT (Import Address Table)
- 프로그램은 `kernel32.dll` 같은 외부 DLL의 함수를 직접 위치를 몰라도 호출할 수 있다.
- 이는 프로그램 내부에 있는 **IAT(임포트 주소 테이블)**, 즉 "함수 이름 → 실제 메모리 주소"가 적힌 표 덕분이다.
- 비유: 전화번호부. 이름으로 전화를 걸면 번호부에 적힌 실제 번호로 연결된다.

### 2. IAT 후킹
- IAT 표의 주소값을 몰래 다른 주소(내가 만든 함수)로 바꿔치기하는 기법.
- 특별한 커널 권한 없이, 유저모드에서 `VirtualProtect()`로 메모리 보호를 풀고 값만 덮어쓰면 가능.
- **한계**: `GetProcAddress()`로 함수 주소를 직접 얻어 IAT를 거치지 않고 호출하면 우회당함.

```
[정상]   IAT["CreateFileW"] = 0x7FFA1234 (진짜 함수)
[후킹 후] IAT["CreateFileW"] = 0x00401000 (내 함수) → 로그 후 진짜 함수 호출(선택)
```

### 3. 트램폴린(인라인) 후킹
- IAT가 아니라 **함수 코드 자체의 시작 몇 바이트**를 `jmp 내코드` 로 덮어쓰는 기법.
- 원본 함수의 앞부분 코드는 다른 곳(트램폴린)에 백업해두고, 내 후킹 함수 실행 후 백업된 코드를 실행 → 원본 함수의 나머지로 다시 점프.
- 함수 이름을 어떻게 찾아왔든(IAT 경유든 직접 주소 획득이든) **함수 코드의 첫 줄은 반드시 실행되므로** IAT 후킹보다 우회가 어렵다.
- 이름의 유래: 원본에서 튕겨나가(jmp) → 내 코드 실행 → 다시 튕겨 돌아오는(jmp back) 모양이 트램폴린과 비슷.

```
0x7FFA1234: jmp 0x00401000   ; 원래 코드 첫 5바이트를 덮어씀
...
[내 후킹 함수]
  1) 로그/검사 수행
  2) 백업해둔 원본 코드 실행 (트램폴린)
  3) 원본 함수의 나머지 부분으로 점프 → 정상 흐름 복귀
```

### 4. 비교표

| 항목 | IAT 후킹 | 트램폴린(인라인) 후킹 |
|---|---|---|
| 건드리는 대상 | 함수 주소 표 | 함수 코드 자체 |
| 구현 난이도 | 쉬움 | 어려움 (백업/복원 필요) |
| 우회 난이도 | 쉬움 | 어려움 |
| 실전 사용처 | 간단한 모니터링 도구 | 실제 상용 EDR/AV 엔진 |
| 관련 오픈소스 | - | Microsoft Detours, MinHook |

---

## 🔗 EDR과의 연관성
- **공격자 관점**: 루트킷/뱅킹 악성코드가 자신을 숨기거나 백신 탐지를 우회할 때 API 후킹 기법을 활용.
- **방어자(EDR) 관점**: 상용 EDR/AV의 유저모드 에이전트가 `ntdll.dll`의 `NtCreateFile`, `NtOpenProcess` 등을 트램폴린 후킹하여 모든 프로세스의 파일/프로세스 접근을 실시간 감시.
- **공격 우회 기법**: 공격자가 EDR의 유저모드 후킹을 감지하고 패치되지 않은 원본 syscall을 직접 호출하는 **Direct Syscall / Unhooking** 기법 — MITRE ATT&CK **T1562.001 (Disable or Modify Tools)**.
- **아키텍처 함의**: 유저모드 후킹은 "가볍지만 우회 가능", 커널 콜백(3단계에서 학습 예정, `PsSetCreateProcessNotifyRoutine` 등)은 "느리지만 우회 어려움" → 실전 EDR은 이 둘을 계층적으로 함께 사용.

---

## ⚠️ 자주 하는 오해/실수
1. 후킹에 항상 관리자/커널 권한이 필요하다 → **아님**. 같은 프로세스 내 유저모드 후킹은 일반 권한으로 가능 (다른 프로세스에 주입하려면 별도 권한 필요, 프로세스 인젝션 개념과 연결).
2. 후킹 = 무조건 악성 → **아님**. EDR, APM, 디버거도 정당하게 사용. 기법 자체는 중립적이며 의도/대상이 중요.
3. 트램폴린 후킹이 있으면 IAT 후킹은 무의미하다 → **아님**. 목적에 따라 둘 다 실전에서 사용됨.
